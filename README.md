# About MODSHOP Web Application
# 🧾 MODSHOP — Integrated Project (Year 2)

This project is part of the course **INT221: Information Technology Integrated Project I**, taken during the second year. The objective is to develop a complete **e-commerce system**, following the specifications provided by the instructor—including project topic, testing method, and even API endpoints.

## 📌 Project Overview

**MODSHOP** is a simulated e-commerce platform developed by a team of 3 students. Every member was required to work across all areas: **Frontend**, **Backend**, and **Infrastructure**. The development process was divided into **5 Sprints**, and at the end of each sprint, we conducted a **User Acceptance Test (UAT)** with our advisor.

We used **Jira** to manage our workflow—creating and tracking **PBIs (Product Backlog Items)**, assigning tasks, and monitoring sprint progress to ensure transparency and efficiency throughout the project lifecycle.

All features were tested using **Cypress**, which was pre-configured by the instructor, to demonstrate complete end-to-end functionality of the system.

## ⚙️ Project Highlights

- Built using **RESTful API** architecture
- Full support for **CRUD** operations
- **Dockerized** for consistent CI/CD and deployment
- Utilized **Nginx** for reverse proxy setup
- **Automated testing** using Cypress
- Task management and sprint planning with **Jira**
- Hands-on experience across **Frontend**, **Backend**, and **DevOps**

> 💬 This project will continue into the first semester of Year 3 for further development and feature expansion.

## 🛠️ Tech Stack

- **Frontend**: Vue.js  
- **Backend**: Java Spring Boot  
- **Database**: MySQL  
- **Tools**: Docker, CI/CD, Nginx, Cypress, Jira
